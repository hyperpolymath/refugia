== Problem
Modern security infrastructure suffers from **"The Surveillance Paradox"**:
* **Static Panopticons:** Rigid surveillance that ignores the fluid nature of modern threats.
* **Platform Dependency:** Absolute reliance on centralized cloud vendors for "safety."
* **Artifact Fragmentation:** Security logic, recovery playbooks, and system state buried across 165+ disparate repositories.
* **The Slop Factor:** Redundant, messy code (jQuery, outdated JS) acting as an unmonitored attack vector.

== Solution: The Kea Model
**Kea** is a distributed, high-assurance infrastructure that secures digital assets through **Relentless Curiosity**. Rejecting the "Top-Down" model, Kea employs an **Investigative-Sensor** approach (Kea-Mandible) coupled with **Cognitive-Heuristic** logic (Kea-Wit). It treats infrastructure as a living ecological niche, employing **Moving Target Defence (MTD)** and **Neurosymbolic Verification** to ensure total system integrity.

== Key Design Concepts

=== The Lithospheric Refugium
Your state is not stored in a "Cloud." It is preserved in **Refugia** (Safe Havens). In the event of an environmental catastrophe (a breach) on **The Range**, the **Kea-Bivouac** (The Roost) uses the immutable DNA stored in Refugia to "re-hatch" a clean instance.



=== Kea-Wit Heuristics
Kea-Wit uses "Wit" (intelligent logic) rather than simple pattern matching.

----
% Logtalk heuristic for prying into files
prying_investigation(File) :-
    kea_mandible::scan(File, Findings),
    wit_engine::evaluate(Findings, Risk),
    Risk > threshold,
    bivouac::trigger_playbook(isolate_and_verify).
